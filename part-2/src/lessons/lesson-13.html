Lesson 13 Part 2: Advanced

Timelines Best Practices

Structure complex animations with master timelines and nested sequences.

Demo - Choreographed Animation

The Code

JavaScript

JavaScript 
import { gsap } from 'gsap';

const boxes = [
  document.querySelector('#box1'),
  document.querySelector('#box2'),
  document.querySelector('#box3')
];

// Master timeline with nested sequences
const master = gsap.timeline({ paused: true });

// Step 1: Fade in boxes sequentially
master.to(boxes, {
  opacity: 1,
  stagger: 0.2,
  duration: 0.5
});

// Step 2: Bounce effect
master.to(boxes, {
  y: -30,
  duration: 0.3,
  stagger: 0.1,
  ease: 'power2.out'
}, '-=0.2');

master.to(boxes, {
  y: 0,
  duration: 0.4,
  stagger: 0.1,
  ease: 'bounce.out'
});

document.querySelector('#playBtn').addEventListener('click', () => master.play());
document.querySelector('#resetBtn').addEventListener('click', () => {
  master.restart().pause();
  gsap.set(boxes, { opacity: 0, y: 0 });
});

How It Works

Timelines are the backbone of complex animations. Use labels and relative positioning to choreograph multi-step sequences.

Best Practices

  • Nested timelines: Break complex animations into reusable pieces
  • Relative positioning: Use += or -= to position relative to end
  • Labels: Name important points with .addLabel()
  • Call callbacks: Use .call() for side effects

Position Parameter

The 3rd parameter in .to() controls timing. Use numbers for absolute time, strings like '-=0.5' for relative.
