JavaScript

JavaScript 
import { gsap } from 'gsap';

// Method 1: CSS Selector (all matching elements)
gsap.to('.boxes-grid .demo-box', {
  x: 0,
  y: 0,
  opacity: 1,
  scale: 1,
  duration: 0.5,
  stagger: 0.05,
  ease: "power2.out"
});

How It Works

GSAP tweens can accept multiple targets - a single animation affects all of them. Combined with stagger, this becomes incredibly powerful.

Three Ways to Select Multiple Elements

1. CSS selector string: '.className'
2. NodeList: document.querySelectorAll('.class')
3. Array: [el1, el2, el3]

Key Points

  • All targets animate identically but independently
  • Each element tracks its own progress
  • Stagger creates offset timing between targets
  • The animation completes when the last element finishes

Performance

GSAP is highly optimized. Animating 100 elements is just as smooth as animating 1. However, keep in mind:

  • More elements = more DOM queries = slower initial setup
  • Complex selectors are slower than stored references
  • If animating hundreds of elements frequently, consider using canvas

Practical Uses

  • Card grid animations
  • List item entrances
  • Gallery/masonry layouts
  • Dashboard widget reveals

Try It Yourself

Edit /src/lessons/lesson-10.js and experiment:

  • Change stagger: Try 0.1 or 0.2 for different speeds
  • Different animation: Make them rotate instead of slide
  • Reverse order: Add stagger: { each: 0.05, from: "end" }
  • Remove stagger: See how they all move together vs in sequence
