Timeline Nesting

Build complex animations from reusable timeline building blocks

JavaScript

JavaScript 
import { gsap } from 'gsap';

// Create a reusable child timeline (paused, so it doesn't play yet)
const boxAnimation = gsap.timeline({ paused: true });
boxAnimation
  .to('[data-box="1"]', { x: 0, opacity: 1, duration: 0.5 })
  .to('[data-box="2"]', { x: 0, opacity: 1, duration: 0.5 });

// Create master timeline that plays the child timeline
const master = gsap.timeline();
master
  .add(boxAnimation.play())
  .to('[data-box="1"]', { scale: 1.2, duration: 0.3 });

How It Works

Nesting means adding one timeline inside another. This lets you create modular, reusable animation sequences that can be combined in different ways.

Key Concept

A child timeline can be added to a parent timeline as a single unit. The parent controls when the child starts playing.

Benefits of Nesting

  • Reusability: Create animation sequences once, use them many times
  • Organization: Break complex animations into manageable pieces
  • Flexibility: Combine timelines in different orders
  • Testing: Debug smaller timelines independently

Position Parameter

When adding a nested timeline, you can control its position:

Position Meaning
"start" At timeline start (default)
"<" At current position
"+=0.5" 0.5 seconds after current
"-=0.5" 0.5 seconds before current

The pattern

Create timelines with paused: true, then use .add(timeline.play()) to incorporate them into parent timelines.
