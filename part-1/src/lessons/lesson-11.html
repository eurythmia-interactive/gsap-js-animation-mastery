JavaScript

JavaScript 
import { gsap } from 'gsap';

const box = document.querySelector('.demo-box');
const status = document.querySelector('#callback-status');
const progressBar = document.querySelector('#progress-bar');

const tween = gsap.to(box, {
  x: 300,
  duration: 2,
  ease: "power2.out",

  // Lifecycle callbacks
  onStart: () => {
    status.textContent = '▶️ Started';
    status.style.color = 'var(--accent)';
  },
  onUpdate: () => {
    const progress = Math.round(tween.progress() * 100);
    progressBar.style.width = `${progress}%`;
    status.textContent = `▶️ Running: ${progress}%`;
  },
  onComplete: () => {
    status.textContent = '✅ Complete!';
    status.style.color = '#10b981';
  }
});

document.querySelector('#restart-btn').onclick = () => {
  tween.restart();
  progressBar.style.width = '0%';
  status.textContent = '🔄 Restarting...';
  status.style.color = 'var(--text-muted)';
};

How It Works

Callbacks are functions that run automatically at specific points during an animation's lifecycle. They let you execute custom code in response to animation events.

Available Callbacks

onStart - Fires when animation begins
onUpdate - Fires every frame during animation
onComplete - Fires when animation finishes
onReverseComplete - Fires when reversed to start
onRepeat - Fires after each repeat

When to Use Callbacks

  • onStart: Initialize something before animation, show loading state
  • onUpdate: Update UI elements reflecting animation progress
  • onComplete: Trigger next animation, hide loading, show success
  • onRepeat: Count repetitions, change animation each cycle

Important Notes

  • Callbacks are functions, so don't call them: use onComplete: myFn not onComplete: myFn()
  • In callbacks, this refers to the tween object
  • onUpdate runs every frame - keep the code efficient!
