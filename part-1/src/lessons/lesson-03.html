gsap.fromTo() - Full Control

Define both WHERE animations start AND where they end

The Code

HTML

HTML 
<section class="section--demo">
  <div class="demo-area">
    <div class="demo-box" style="opacity: 0; transform: translateY(100px) rotate(-45deg);"></div>
  </div>
</section>

CSS

CSS 
.demo-box {
  width: 100px;
  height: 100px;
  background: #ff6b35;
  border-radius: 12px;
  box-shadow: 0 0 30px rgba(255, 107, 53, 0.5);
}

JavaScript

JavaScript 
import { gsap } from 'gsap';

const box = document.querySelector('.demo-box');

// gsap.fromTo() takes 3 arguments: target, fromVars, toVars
gsap.fromTo(box,
  {
    // FROM: Starting values
    opacity: 0,
    y: 100,
    rotation: -45
  },
  {
    // TO: Ending values (includes timing)
    opacity: 1,
    y: 0,
    rotation: 0,
    duration: 1,
    ease: "power2.out"
  }
);

How It Works

gsap.fromTo() gives you complete control over both the starting and ending values of an animation. It takes three arguments:

Syntax

gsap.fromTo(target, fromVars, toVars)

target - Element to animate
fromVars - Starting values
toVars - Ending values (must include timing: duration, ease)

Why Use fromTo()?

  • When you need to specify both start and end precisely
  • For complex entrance/exit animations
  • When to() or from() alone aren't enough

Comparison with to() and from()

Method Start Point End Point
gsap.to() Current state Your values
gsap.from() Your values Current state
gsap.fromTo() Your values Your values

Important

Remember that toVars (the third argument) must include your timing properties like duration and ease. Without them, the animation won't know how long to take!
