gsap.set() - Instant Animation

Set properties instantly without any animation

The Code

HTML

HTML 
<section class="section--demo">
  <div class="demo-area">
    <div class="demo-box" style="opacity: 0; transform: translateX(200px) scale(0.5);"></div>
  </div>
</section>

CSS

CSS 
.demo-box {
  width: 100px;
  height: 100px;
  background: #ff6b35;
  border-radius: 12px;
  box-shadow: 0 0 30px rgba(255, 107, 53, 0.5);
}

JavaScript

JavaScript 
import { gsap } from 'gsap';

const box = document.querySelector('.demo-box');

// gsap.set() changes properties INSTANTLY (no duration)
gsap.set(box, {
  x: 200,
  opacity: 0,
  scale: 0.5
});

// Then animate with gsap.to()
gsap.to(box, {
  x: 0,
  opacity: 1,
  scale: 1,
  duration: 1,
  ease: "power2.out"
});

How It Works

gsap.set() is like gsap.to() but with no duration - properties change instantly. Think of it as setting the initial state before an animation.

Key Difference

gsap.set() = Instant (duration: 0)
gsap.to() = Animated (duration: any positive number)

Common Use Cases

  • Set initial hidden state: Make elements invisible before animating them in
  • Establish starting positions: Place elements off-screen before sliding them in
  • Reset elements: Return elements to their starting position before replaying
  • Configure transforms: Set transform-origin for rotation animations

Why Not Just Use CSS?

While CSS can set initial states, gsap.set() is useful when:

  • You want all your animation logic in JavaScript
  • You need to dynamically calculate starting positions
  • You're setting up a timeline and want explicit starting points

Important

gsap.set() doesn't accept timing properties like duration, delay, or ease because there's no animation - it's instantaneous!
