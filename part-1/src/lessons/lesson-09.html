JavaScript

JavaScript 
import { gsap } from 'gsap';

const boxes = document.querySelectorAll('.demo-box');

// Each box has the same animation but different easing
boxes.forEach((box, i) => {
  const easings = ['linear', 'power1.out', 'power2.out', 'power3.out', 'bounce.out'];
  
  gsap.to(box, {
    x: 400,
    duration: 1.5,
    ease: easings[i],
    delay: i * 0.3
  });
});

How It Works

Easing controls HOW an animation progresses through time. Without easing, motion feels robotic. With easing, motion feels natural and organic.

Visual Analogy

Think of easing like a car's acceleration:
Linear = maintaining exactly 50mph the whole way
power2.out = starts fast (accelerating from 0) then slows to stop

Easing Families

Family Feel Best For
power1-4 Increasing deceleration General entrances
linear Constant speed Progress bars, loading
back Overshoots then settles Playful UI elements
elastic Springy oscillation Attention grabbers
bounce Bouncy landing Physical objects

The ".out", ".in", ".inOut" Suffixes

  • .out - Fast start, slow end (like a car stopping)
  • .in - Slow start, fast end (like a car starting)
  • .inOut - Slow start and end (like a car stopping and starting)

Parameters

Some easings accept parameters: "back.out(1.7)" or "elastic.out(1, 0.5)". The numbers control the intensity of the effect.
