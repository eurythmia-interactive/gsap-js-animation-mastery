gsap.fromTo() - Full Control

Define both WHERE animations start AND where they end

The Code

HTML

HTML 
<section class="section--demo">
  <div class="demo-area">
    <div class="demo-box" style="opacity: 0; transform: scale(0) rotate(-180deg);"></div>
  </div>
</section>

CSS

CSS 
.demo-box {
  width: 100px;
  height: 100px;
  background: #ff6b35;
  border-radius: 12px;
  box-shadow: 0 0 30px rgba(255, 107, 53, 0.5);
}

JavaScript

JavaScript 
import { gsap } from 'gsap';

const box = document.querySelector('.demo-box');

// gsap.fromTo() - define both start AND end values
gsap.fromTo(box,
  {
    opacity: 0,
    scale: 0,
    rotation: -180
  },
  {
    opacity: 1,
    scale: 1,
    rotation: 0,
    duration: 1,
    ease: "back.out(1.7)"
  }
);

How It Works

gsap.fromTo() gives you complete control - you define both WHERE the animation starts AND where it ends.

The Syntax

gsap.fromTo(target, fromVars, toVars)

fromVars - Starting values
toVars - Ending values (includes duration & ease)

Visual Comparison

gsap.fromTo() [START: fromVars] ──────────► [END: toVars]

Why Use fromTo()?

  • Define both start and end explicitly
  • No confusion about what "current state" means
  • Perfect for entrance animations with precise control

The back.out Easing

In this example we used "back.out(1.7)" which creates an overshoot effect - the element briefly goes past its target then settles back. The number (1.7) controls how much it overshoots.

Try It Yourself

Edit /src/lessons/lesson-02.js and experiment:

  • Different start values: Try scale: 2 (starts extra large)
  • Vertical motion: Add y: -100 to animate from above
  • Change easing: Try "elastic.out" or "bounce.out"
  • Rotation: Try different starting rotations like 90 or 360
