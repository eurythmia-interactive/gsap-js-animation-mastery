gsap.to() - Your First Animation

Learn the basic syntax of GSAP and how tweens work

The Code

HTML

HTML 
<section class="section--demo">
  <div class="demo-area">
    <div class="demo-box" style="opacity: 0; transform: translateX(100px);"></div>
  </div>
</section>

CSS

CSS 
.demo-box {
  width: 100px;
  height: 100px;
  background: #ff6b35;
  border-radius: 12px;
  box-shadow: 0 0 30px rgba(255, 107, 53, 0.5);
}

JavaScript

JavaScript 
import { gsap } from 'gsap';

// Select the box element
const box = document.querySelector('.demo-box');

// Animate FROM current state TO the values you specify
gsap.to(box, {
  opacity: 1,
  x: 0,
  duration: 1,
  ease: "power2.out"
});

How It Works

gsap.to() is the most fundamental method in GSAP. It takes an element's current state and animates it TO the values you specify.

The Core Concept

gsap.to(target, { properties }) means: "Animate the target FROM where it currently is TO these new values."

Syntax Breakdown

  • target - The element(s) to animate (selector string, element, or array)
  • properties - An object containing CSS properties and their target values

Common Properties

Property Description Example
duration Animation length in seconds 1 = 1 second
ease Motion curve "power2.out"
delay Wait before starting 0.5 = half second

Transform Properties (Use These!)

GSAP uses CSS transforms which are GPU accelerated for smooth performance:

  • x - Horizontal movement (px)
  • y - Vertical movement (px)
  • scale - Size multiplier (1 = 100%)
  • rotation - Degrees of rotation

Why "power2.out"?

The ease property controls the feeling of the motion. "power2.out" means "fast start, slow finish" - the most natural feeling for entrance animations. We'll explore easing in detail in Lesson 09.

Try It Yourself

Edit /src/lessons/lesson-01.js and experiment:

  • Change duration: Try 0.5 (faster) or 2 (slower)
  • Different easing: Try "bounce.out" or "elastic.out"
  • Add rotation: Add rotation: 360 to spin the box
  • Scale up: Add scale: 1.3 to enlarge the box
  • Add delay: Add delay: 1 to wait before starting
