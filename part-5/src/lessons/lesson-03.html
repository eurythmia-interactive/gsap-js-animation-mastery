Lesson 03 Part 5: Landing Page Design

Responsive Typography

Your landing page typography must adapt across screen sizes. Using Tailwind's responsive prefixes and fluid sizing techniques, you can create type that scales beautifully from mobile to ultrawide displays.

The Code

Responsive Text Sizes

HTML 
<!-- Mobile: text-3xl, Tablet: text-4xl, Desktop: text-5xl, Large: text-6xl -->
<h2 class="text-3xl sm:text-4xl md:text-5xl lg:text-6xl font-bold font-display">
  Responsive Headline
</h2>

<!-- Fluid Typography with clamp() -->
<h1 class="text-[clamp(2rem,5vw,4rem)] font-bold">
  Fluid Headline
</h1>

How It Works

Responsive Prefixes

  • text-3xl → Base (mobile), no prefix
  • sm:text-4xl → 640px and up
  • md:text-5xl → 768px and up
  • lg:text-6xl → 1024px and up
  • xl:text-7xl → 1280px and up
  • 2xl:text-8xl → 1536px and up

Breakpoint Reference

  • Base - 0px (Mobile)
  • sm: - 640px (Large phones)
  • md: - 768px (Tablets)
  • lg: - 1024px (Laptops)
  • xl: - 1280px (Desktops)
  • 2xl: - 1536px (Large screens)

Fluid Typography with clamp()

For smooth scaling without discrete breakpoints:

text-[clamp(2rem, 5vw, 4rem)]

This scales from 2rem to 4rem smoothly based on viewport width.

Best Practice

  • Start with mobile, then scale up
  • Use leading-tight for large headlines
  • Reduce tracking on larger sizes
  • Test on actual devices when possible
